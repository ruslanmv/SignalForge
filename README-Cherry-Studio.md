## üì¶ Step 2: Get Project Code

**Why do I need to get the project code?**

The AI analysis function needs to read the news data within the project to work. Whether you use GitHub Actions or Docker for deployment, the news data generated by the crawler is saved in the project's `output` directory. Therefore, before configuring the MCP server, you must obtain the complete project code (including data files).

Choose one of the following methods based on your technical level:

### Method 1: Git Clone (Recommended for technical users)

If you are familiar with Git, you can use the following command to clone the project:

```bash
git clone https://github.com/ruslanmv/SignalForge.git
cd SignalForge
```

**Advantage**:

  - You can update local data to the latest version with a single command (`git pull`).

### Method 2: Download ZIP Directly (Recommended for beginners)

1.  **Visit the GitHub Project Page**

      * Project Link: `https://github.com/ruslanmv/SignalForge`

2.  **Download the Compressed File**

      * Click the green **"Code"** button.
      * Select **"Download ZIP"**.
      * Or visit directly: `https://github.com/ruslanmv/SignalForge/archive/refs/heads/master.zip`

**Note**:

  - This method is slightly more tedious; updating data later requires repeating the steps above and overwriting your local data (the `output` directory).

## üöÄ Step 3: One-Click MCP Server Deployment

### Windows Users

1.  **Double-click** to run `setup-windows.bat` in the project folder.
2.  **Wait** for the installation to complete.
3.  **Record the displayed configuration information** (command path and arguments).

### Mac Users

1.  **Open Terminal** (search for "Terminal" in Launchpad).
2.  **Drag and drop** the `setup-mac.sh` file from the project folder into the terminal window.
3.  **Press Enter**.
4.  **Record the displayed configuration information**.
